## How it works

<CardGrid>
  <Card title="1. Define" icon="pencil">
    Write your schema in plain SQL files. Tables, indexes, functions, policies — everything in native PostgreSQL DDL.
  </Card>
  <Card title="2. Diff" icon="magnifier">
    pgmold introspects your live database, compares it to your SQL files, and computes the exact delta.
  </Card>
  <Card title="3. Apply" icon="rocket">
    Apply the migration in a single transaction. Destructive operations are blocked unless you explicitly opt in.
  </Card>
</CardGrid>

```bash
$ pgmold plan -s sql:schema.sql -d postgres://localhost/mydb

-- Migration Plan (3 operations)
ALTER TABLE "public"."users" ADD COLUMN "email" TEXT NOT NULL;
CREATE INDEX "users_email_idx" ON "public"."users" ("email");
CREATE POLICY "users_read_own" ON "public"."users" FOR SELECT USING (auth.uid() = id);
```

## Features

<CardGrid stagger>
  <Card title="Native SQL" icon="document">
    No HCL, no YAML, no DSLs. Write standard PostgreSQL DDL. If Postgres accepts it, pgmold understands it.
  </Card>
  <Card title="Drift detection" icon="warning">
    SHA256 fingerprint comparison catches manual schema changes. Use in CI to prevent drift.
  </Card>
  <Card title="Safety linting" icon="approve-check-circle">
    Destructive operations blocked by default. Lock hazard warnings prevent downtime. Production mode blocks table drops entirely.
  </Card>
  <Card title="RLS policies" icon="setting">
    Row-Level Security policies are first-class citizens. Define, diff, and apply policies alongside your tables.
  </Card>
  <Card title="Multi-schema" icon="list-format">
    Manage auth, api, public, and custom schemas. Cross-schema foreign keys handled automatically.
  </Card>
  <Card title="CI/CD ready" icon="github">
    GitHub Action included. JSON output for pipelines. Non-zero exit on drift. PR comments with migration plans.
  </Card>
  <Card title="Transactional apply" icon="approve-check">
    All migrations run in a single transaction. If anything fails, nothing changes.
  </Card>
  <Card title="Terraform provider" icon="puzzle">
    Manage PostgreSQL schemas as infrastructure-as-code with the pgmold Terraform provider.
  </Card>
  <Card title="ORM integration" icon="random">
    Use Drizzle ORM schemas as source. Mix SQL and ORM sources.
  </Card>
</CardGrid>
